Опишите отличия между командами rebase и merge в git. Когда стоит использовать rebase?

Git Merge

Слияние принимает содержимое ветки источника и объединяет их с целевой веткой. В этом процессе изменяется только целевая ветка. История исходных веток остается неизменной. Плюсы: простота; сохраняет полную историю и хронологический порядок; поддерживает контекст ветки. Минусы: история коммитов может быть заполнена (загрязнена) множеством коммитов; отладка с использованием git bisect может стать сложнее.

Git Rebase

Rebase сжимает все изменения в один «коммит». Затем интегрирует коммиит в ветку. В отличие от merge, rebase перезаписывает историю, потому что она передает завершенную работу из одной ветки в другую. В процессе устраняется нежелательная история.

Rebase стоит использовать тогда, когда полностью завершена работа над каким либо модулем, но за 3 года разработки в команде из 20 человек, мы не разу не использовали данную фичу т.к историю изменений лучше сохрянять если нет 200% уверенности в готовности модуля
